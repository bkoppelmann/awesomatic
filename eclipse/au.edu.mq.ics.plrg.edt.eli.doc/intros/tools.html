Tools

This section of the Eli documentation contains descriptions of tools and associated specification notations used by the Eli system.

The FunnelWeb manual describes how to use Literate Programming to specify Eli specifications in coherent, documented modules.

The Pattern-based Text Generator manual shows how to describe structured output patterns in a flexible way for use by other parts of the language processor.

The Definition Table manual shows how to represent entities used by the language processor and their properties.

The OIL manual describes how to specify operator identification and type analysis problems in a way that can be used in implementations of programming language compilers.

The Tree Parsing manual describes a language for transforming, interpreting and printing tree-structured data.

The Command-line Processing manual describes a language for specifying the format of a language processor's command line and methods for using the command-line information in other parts of the processor.

The Tree Unparsing manual describes how to produce unparsers that process a tree structure to produce some form of structure output text.

The Execution Monitoring manual shows how to use Eli's Noosa tool to perform specification-level debugging and analysis of an executing language processor.

Option languages

The Cola Options manual describes options that control the Cola parser generator which is one of the parser generator options available in Eli.

The LIGA Control Language manual describes options that can be used to control the operation of the LIGA tool which processes the LIDO Tree Computation language.

The LIGA Show manual describes ways in which Eli can be asked to provide useful information about the LIDO specifications presented to Eli.

The GORTO manual describes a tool that can be used to analyse the attribution computation dependencies implied by LIDO specifications.
